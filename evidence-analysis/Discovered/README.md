# Discovered Architectures

Automatically discovered architecture catalog entries produced by the SPEC-002 Dataset Scaling Pipeline. Each entry is generated by running `extract-signals.sh` (SPEC-001) against a public GitHub repository and classifying the output using the heuristic classifier.

## Relationship to other evidence sources

| Source | Evidence type | How produced |
|--------|--------------|--------------|
| **ReferenceArchitectures** | reference-implementation | Manually curated, canonical examples |
| **TheKataLog** | competition-submission | Design-only kata submissions |
| **AOSA** | book-chapter | Narrative descriptions from book chapters |
| **RealWorldASPNET** | production-application | Production .NET applications |
| **Discovered** (this source) | automated-discovery | Batch pipeline signal extraction + heuristic classification |

## Directory structure

```
Discovered/
├── README.md              (this file)
├── _index.yaml            (aggregated index of all discovered projects)
├── quality-report.md      (pipeline quality report)
└── docs/
    ├── catalog/           (one YAML file per discovered project)
    │   ├── _index.yaml    (catalog-level index)
    │   └── *.yaml         (individual catalog entries)
    └── analysis/          (cross-source analysis documents)
```

## Quality gating

- Entries with `discovery_metadata.confidence >= 0.5` are automatically accepted
- Entries with `confidence < 0.5` are flagged with `review_required: true`
- All entries include full signal breakdown for traceability

## Pipeline

See `pipeline/` in the repository root for the batch discovery pipeline (manifest, orchestrator, classifier, report generator).
