- **Single codebase, single build**: The presence of `go.mod` at the repository root indicates a single Go module. The `nx.json` and `project.json` files indicate a unified monorepo build system (Nx) managing both frontend and backend assets, not independent services.

- **Unified packaging**: The `packaging/` directory contains scripts for building a single binary/docker image across multiple platforms (deb, rpm, docker, mac), rather than packaging separate services. The `embed.go` file at the root suggests static asset embedding into a single binary.

The `devenv/docker/` directory contains development environment setups for external dependencies (databases, load testing), not internal service decomposition.

- **Plugin types**: The `e2e-playwright/test-plugins/` directory contains distinct plugin types: `grafana-test-datasource` (data source plugins), `grafana-test-panel` (panel plugins), and `grafana-extensionstest-app` (app plugins). - **Plugin sandboxing**: The presence of `frontend-sandbox-app-test`, `frontend-sandbox-datasource-test`, and `frontend-sandbox-panel-test` directories indicates a sandboxed plugin execution environment, a key characteristic of Microkernel architectures where extensions are isolated from the core.

You can specify a data source on a per-query basis. This works for even custom datasources." This indicates the core provides extension APIs for custom implementations.
