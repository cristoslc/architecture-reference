## Summary

Supabase Storage is a production-grade, multi-tenant object storage service built as a single deployable TypeScript/Node.js application using the Fastify framework. It exposes multiple wire protocols (HTTP REST, S3-compatible API, TUS resumable uploads, Apache Iceberg REST catalog, and a vector endpoint) all registered as route modules within a single Fastify instance. The core domain logic is organized into a clear horizontal layering of HTTP presentation, storage orchestration, and infrastructure concerns, with the storage backend and database layers sitting behind abstract adapter interfaces (Ports and Adapters / Hexagonal design). Background work (webhook dispatch, object lifecycle events, tenant migrations, Iceberg catalog reconciliation) is handled asynchronously via a PostgreSQL-backed pg-boss job queue. Multi-tenancy is implemented at the application layer using a shared PostgreSQL database with per-tenant Row Level Security policies, connection pooling, and a pub/sub cache-invalidation mechanism for tenant configuration changes. The system can be deployed as a combined API+worker process or as separate server and worker processes from the same codebase.

## Quality Attributes

- **Scalability**: Multi-tenant architecture with per-tenant connection pools, adaptive pool sizing, and cluster-aware rebalancing (ECS/EKS integration)
- **Extensibility**: Pluggable storage backends (S3, local file) and database adapters via abstract interfaces; new backends require only implementing the adapter contract
- **Observability**: OpenTelemetry tracing, Prometheus metrics, Logflare integration, structured pino logging, per-tenant feature flags
- **Reliability**: pg-boss dead-letter queues and 20-retry limits for background jobs; graceful shutdown with `AsyncAbortController` coordinating all subsystems; database transactions and advisory locks for destructive operations
- **Security**: JWT validation with JWKS support, AWS Signature V4 request signing, Row Level Security via PostgreSQL, `asSuperUser()` escalation path for admin operations, rate limiting via Fastify plugin
- **Testability**: Dependency injection throughout; hexagonal design isolates business logic from infrastructure; dedicated `src/internal/testing/` module
- **Multi-tenancy**: Shared PostgreSQL with RLS; per-tenant config cache with pub/sub invalidation; tenant-scoped feature flags; separate multitenant admin server
