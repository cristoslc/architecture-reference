## Architecture Styles

### Primary: Multi-Agent

MetaGPT's entire design is organised around autonomous agents (Roles) that perceive messages, reason, and act in a shared Environment. The `Role` base class (`metagpt/roles/role.py`) gives every agent identity (`name`, `profile`, `goal`), private memory (`Memory`, `working_memory`), a cognition loop (`_observe` → `_think` → `_act`), and a pluggable react mode (`REACT`, `BY_ORDER`, `PLAN_AND_ACT`). Concrete role subclasses — `ProductManager`, `Architect`, `Engineer`, `QaEngineer`, `DataInterpreter`, `TeamLeader`, `SWEAgent` — are independently instantiated and hired into a `Team` via `team.hire(roles)`. The `Environment` and its `MGXEnv` variant act as a shared message bus that routes typed `Message` objects between agents using address-set subscription (`role.set_addresses`), so agents remain decoupled while still reacting to each other's outputs. Coordination follows an explicit SOP pattern ("`Code = SOP(Team)`"): each role watches the `cause_by` tag of the actions it depends on, creating a structured workflow without hard-coded call chains.

### Secondary: Pipeline (Pipe-and-Filter)

Within the standard software-development workflow, the agent team executes a deterministic sequential pipeline: `UserRequirement` → `WritePRD` (ProductManager) → `DesignAPI` (Architect) → `WriteTasks` (ProjectManager) → `WriteCode` (Engineer) → `WriteCodeReview` (Engineer) → `SummarizeCode`. Each stage produces a typed `Message` (with `cause_by` pointing to the producing `Action`), which is placed into the global environment where the next-stage role picks it up by watching for that exact `cause_by` type. The `Researcher` role makes this pipeline structure explicit: it uses `RoleReactMode.BY_ORDER` and `CollectLinks → WebBrowseAndSummarize → ConductResearch` in strict order. The `ActionGraph` class (`metagpt/actions/action_graph.py`) provides a topological-sort harness for dependency-ordered action execution within more complex flows.

## Domain

AI agent framework that models a software company as a multi-agent system. Given a one-line user requirement, MetaGPT orchestrates product manager, architect, project manager, engineer, and QA engineer agents to produce user stories, system design, code, and tests. Also supports data analysis (DataInterpreter), research automation (Researcher), SWE-agent style code repair (SWEAgent), and interactive conversational teams (MGX / mgx.dev).

## Production Context

- Shipped as a pip package (`metagpt`) with a CLI entry point (`metagpt --help`); also powers the cloud product mgx.dev (Product Hunt #1 of the Week, March 2025).
- Supports 15+ LLM backends via the `provider/` abstraction layer — GPT-4, Claude, Gemini, Ollama, Bedrock, Qianfan, and more.
- Single-process deployment (no Docker or microservice infrastructure required); state is persisted to a local project directory with git archiving.
- The `metagpt/ext/` tree contains research-grade extensions (AFlow, SELA, SPO, stanford_town, werewolf game simulation) packaged within the same repo — indicating an academic/research lineage alongside the production product.
