## Style Rationales

**Microkernel (primary, 0.90):** The `blocksuite/` directory implements a formal extension system (`ExtensionType`, `BaseExtensionProvider`, `ExtensionManager`) where the core framework provides minimal runtime (CRDT sync, DI container, lifecycle hooks) and all block types (paragraph, image, database, surface, etc.) register as independent plugins. The backend mirrors this via `AppModuleBuilder` with conditional module loading (`useIf()`) based on deployment flavor, making entire server capabilities opt-in plugins.

**Modular Monolith (secondary, 0.85):** `packages/common/infra/src/framework/` provides a custom DI framework with explicit component types (Service, Entity, Store, Scope) and a hierarchical scope system (Root → Workspace → Doc). ~50 modules in `packages/frontend/core/src/modules/` are wired together at startup but deploy as a single unit per platform (web, Electron, iOS, Android). The NestJS backend deploys as one container (`ghcr.io/toeverything/affine`) with all enabled modules co-located.

**Event-Driven (secondary, 0.72):** A typed `EventBus` built on `eventemitter2` propagates domain events across modules (e.g., `workspace.members.invite`, `workspace.members.removed`). BullMQ queues (nightly, notification, doc, copilot, indexer) decouple async work. The event bus also supports cluster-wide broadcast via Socket.io `serverSideEmit`, enabling loose coupling between server instances.

## Domain

Knowledge management and collaborative workspace platform (Notion + Miro alternative). Core domains: document editing (CRDT-based), whiteboard/canvas, real-time collaboration, AI copilot, user/workspace management, payment/quota.

## Production Context

- Self-hostable via Docker and Helm; cloud offering at `app.affine.pro`; Electron desktop and mobile (iOS/Android) clients
- Backend deployed as four independent server flavors (graphql, front/allinone, sync, doc) with separate Helm charts and replica counts
- CRDT synchronization via Yjs/y-octo enables local-first operation; real-time sync over WebSocket (Socket.io rooms per workspace/doc)
- Multi-AI-provider copilot (OpenAI, Anthropic, Gemini, Fal, Perplexity, Morph) with MCP controller at `WorkspaceMcpController`
