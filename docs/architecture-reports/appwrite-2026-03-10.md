## Style Rationales

**Modular Monolith (primary, 0.92):** The entire backend is built from a single PHP codebase (`appwrite-dev` image) deployed in multiple roles based on entrypoint — `app/http.php` (HTTP API), `app/realtime.php` (WebSocket server), `app/worker.php` (queue workers). The `Appwrite` platform class in `src/Appwrite/Platform/Appwrite.php` orchestrates eight domain modules registered via `addModule()`: Account, Databases, Functions, Sites, Console, Proxy, Tokens, and Compute. Each module in `src/Appwrite/Platform/Modules/` contains its own `Module.php`, `Http/` route actions, and `Services/` — a consistent vertical slice per bounded domain. All modules share a single MariaDB instance and a single Redis instance, confirming monolith deployment with modular internal structure.

**Event-Driven (secondary, 0.90):** Redis (via Utopia Queue) acts as the message broker for all async processing. The `Event` class in `src/Appwrite/Event/Event.php` declares twelve named queues (`v1-database`, `v1-deletes`, `v1-audits`, `v1-mails`, `v1-functions`, `v1-webhooks`, `v1-certificates`, `v1-builds`, `v1-messaging`, `v1-migrations`, `v1-stats-resources`, `v1-stats-usage`). Every API controller injects typed queue publishers (`queueForEvents`, `queueForFunctions`, `queueForWebhooks`, `queueForRealtime`, `queueForMails`, `queueForStatsUsage`) and dispatches work asynchronously post-HTTP response. Ten dedicated worker containers (`appwrite-worker-*`) subscribe to specific queues and process events independently. The `appwrite-realtime` container implements a WebSocket push channel, subscribing to PubSub events and broadcasting them to connected clients via `src/Appwrite/PubSub/`.

## Domain

Backend-as-a-Service (BaaS) platform for web, mobile, and native applications. Core service domains: authentication/accounts, databases, file storage, cloud functions (serverless), real-time subscriptions, messaging/push notifications, VCS-integrated builds, SSL certificate management, project administration.

## Production Context

- Self-hosted via Docker Compose; Appwrite Cloud offering at `cloud.appwrite.io`
- All backend containers run from a single `appwrite-dev` image differentiated by entrypoint — this is a modular monolith, not microservices
- Functions executions are delegated to an external `openruntimes/executor` sidecar that launches per-runtime Docker containers, providing sandboxed serverless execution within an otherwise monolithic system
- Redis serves dual duty as both message broker (Utopia Queue) and PubSub channel (for real-time WebSocket fanout)
- Scheduled tasks (`appwrite-task-maintenance`, `appwrite-task-scheduler-*`) run as separate containers on cron schedules alongside the event-driven workers
