## Architecture Styles

### Primary: Modular Monolith

Prowlarr runs as a single ASP.NET Core process (Kestrel, bootstrapped via `NzbDrone.Host/Bootstrap.cs` using DryIoc), deploying one binary per platform. The solution (`Prowlarr.sln`) spans roughly 28 projects — all compiled into one executable. The dominant project is `NzbDrone.Core` (~938 .cs files) which hosts every domain: `Indexers/`, `IndexerSearch/`, `Applications/`, `Download/`, `Notifications/`, `History/`, `HealthCheck/`, `IndexerProxies/`, `Jobs/`, and `Profiles/`. Each domain folder carries its own models, services, repositories, and events — organizing business capability vertically within the single shared process. Persistence is via SQLite (default) or PostgreSQL through a `BasicRepository<T>` (Dapper-based) with 21+ ordered migration scripts under `Datastore/Migration/`.

The core differentiator from a sibling like Lidarr is Prowlarr's role: it is an **indexer manager and proxy** that synchronizes indexer configuration outward to the rest of the \*arr ecosystem (Sonarr, Radarr, Lidarr, Readarr, Mylar, Whisparr) via a dedicated `Applications/` subsystem. The `ApplicationService` handles `ProviderAddedEvent<IIndexer>`, `ProviderUpdatedEvent<IIndexer>`, and `ApplicationIndexerSyncCommand` to push indexer definitions to registered downstream apps over their REST APIs.

### Secondary: Microkernel

The `ThingiProvider` framework (`NzbDrone.Core/ThingiProvider/ProviderFactory.cs`) is a generic runtime plugin bus shared by every extension category. All extension types — indexers, download clients, notifications, indexer proxies, and connected applications — implement `IProvider`, register a `ProviderDefinition`, and are resolved from the DryIoc container at runtime via `_container.GetRequiredService(type)`. The plugin population is large:

- **Indexers**: 71+ definitions in `Indexers/Definitions/`, covering 500+ trackers/newsgroups via native C# classes or the Cardigann YAML-driven engine (`CardigannBase`, `CardigannParser`, `CardigannRequestGenerator`)
- **Download clients**: ~20 implementations (qBittorrent, Transmission, SABnzbd, Deluge, rTorrent, NZBGet, Flood, Aria2, etc.)
- **Notifications**: ~20 targets (Discord, Telegram, Slack, Email, Webhook, Pushover, Apprise, Ntfy, etc.)
- **Indexer proxies**: HTTP, SOCKS4, SOCKS5, FlareSolverr — applied per-indexer
- **Connected applications**: Sonarr, Radarr, Lidarr, Readarr, Mylar, Whisparr, LazyLibrarian — each a dedicated `ApplicationBase` implementation

The internal messaging infrastructure reinforces extensibility: `EventAggregator` (`Messaging/Events/EventAggregator.cs`) dispatches typed events to all `IHandle<TEvent>` (synchronous) and `IHandleAsync<TEvent>` (async via `TaskFactory`) subscribers auto-discovered by the container; `CommandExecutor` dispatches `IExecute<TCommand>` handlers across a `CommandQueue`. Adding a new indexer, proxy, or downstream app integration requires zero changes to the factory core.

## Domain

Media Automation / Indexer Manager & Proxy — an open-source indexer manager that aggregates 500+ torrent trackers and Usenet indexers under a single Newznab/Torznab-compatible API, then syncs those indexer configurations to downstream \*arr apps (Sonarr, Radarr, Lidarr, Readarr, Mylar, Whisparr). Part of the Servarr/\*arr ecosystem.

---

## Production Context

- Ships as a cross-platform .NET process (default port 9696) deployable as a Windows service, Linux daemon, or Docker container.
- Data persistence via SQLite (default) or PostgreSQL; 21+ migration steps.
- The Cardigann YAML engine allows community-maintained tracker definitions without a code change or recompile.
- Per-indexer proxy support (HTTP, SOCKS4, SOCKS5, FlareSolverr) enables bypass of tracker-level anti-scraping measures.
- React/TypeScript SPA frontend communicates over REST API (Swagger at `/docs/v1/openapi.json`) and SignalR for live search and sync status.
- In the \*arr ecosystem Prowlarr occupies a Service-Based integration role — it is a standalone monolith that acts as an indexer aggregation hub for all other \*arr services.
