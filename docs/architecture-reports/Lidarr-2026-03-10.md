## Architecture Styles

### Primary: Modular Monolith

Lidarr is a single deployable ASP.NET Core process (hosted via Kestrel, bootstrapped in `NzbDrone.Host/Bootstrap.cs` using DryIoc). The solution (`Lidarr.sln`) comprises roughly 25 projects organized by technical and cross-cutting concern: `NzbDrone.Core` (1,399 .cs files, all business logic), `Lidarr.Api.V1` (REST controllers), `Lidarr.Http` (middleware, auth), `NzbDrone.SignalR` (real-time push), and `NzbDrone.Common` (shared infrastructure). Within `NzbDrone.Core`, each functional domain — `Music/`, `Download/`, `Indexers/`, `Notifications/`, `MediaFiles/`, `Queue/`, `History/`, `ImportLists/` — is vertically organized with its own models, services, repositories, and events. The system deploys as one process backed by a single SQLite or PostgreSQL database.

### Secondary: Microkernel

The `ThingiProvider` framework (`NzbDrone.Core/ThingiProvider/ProviderFactory.cs`) provides a runtime plugin bus shared by every major extension category. Indexers, download clients, notifications, import lists, and metadata providers all implement `IProvider`, register via `ProviderDefinition`/`ProviderFactory<TProvider,TDef>`, and are resolved from the DryIoc container at request time (`_container.GetRequiredService(type)`). Adding a new download client (there are ~20) or notification target (~30 implementations) requires zero changes to the factory core. The internal messaging layer reinforces this: `EventAggregator` (`Messaging/Events/EventAggregator.cs`) fan-outs typed events to all `IHandle<TEvent>` subscribers discovered by the container, and `CommandExecutor` dispatches typed commands to `IExecute<TCommand>` handlers across three worker threads.

## Domain

Media Automation / Music Library Management — an open-source automated music collection manager that monitors MusicBrainz metadata, searches configured indexers for releases, sends grabs to download clients, and organizes imported audio files into a structured library. Part of the Servarr /*arr ecosystem.

## Production Context

- Ships as a cross-platform .NET process deployable as a Windows service, Linux daemon, or Docker container (default port 8686).
- Data persistence via SQLite (default) or PostgreSQL; 80-step migration history.
- The `ThingiProvider`/plugin system enables runtime addition of indexers (Newznab, Torznab, Redacted, Gazelle), download clients (qBittorrent, Transmission, SABnzbd, etc.), and notification targets (Discord, Email, Webhook, Plex, etc.) from the web UI.
- React/TypeScript SPA frontend communicates over REST API and SignalR for live command/download progress updates.
- Participates in the *arr Service-Based ecosystem: receives indexer sync from Prowlarr and surfaces its API to Overseerr; internally it is a self-contained modular monolith.
