## Architecture Classification: Modular Monolith

### Evidence for Modular Monolith

**Single Deployable with Module Boundaries:**
- The `Bootstrapper` project (`src/Bootstrapper/Confab.Bootstrapper/Confab.Bootstrapper.csproj`) references all modules and serves as the single entry point
- All modules are compiled into one application and run via `dotnet run` from the Bootstrapper

**Explicit Module Contract (`IModule` interface):**
```csharp
// src/Shared/Confab.Shared.Abstractions/Modules/IModule.cs
public interface IModule
{
    string Name { get; }
    string Path { get; }
    IEnumerable<string> Policies => null;
    void Register(IServiceCollection services);
    void Use(IApplicationBuilder app);
}
```

**Dynamic Module Loading with Enable/Disable Support:**
```csharp
// src/Bootstrapper/Confab.Bootstrapper/ModuleLoader.cs
var enabled = configuration.GetValue<bool>($"{moduleName}:module:enabled");
if (!enabled) { disabledModules.Add(file); }
```

**Inter-Module Communication via Local Contracts:**
- `IModuleClient` provides `SendAsync()` for request/response and `PublishAsync()` for broadcast
- Type translation via serialization prevents direct type dependencies between modules:
```csharp
// src/Shared/Confab.Shared.Infrastructure/Modules/ModuleClient.cs
private T TranslateType<T>(object value)
    => _moduleSerializer.Deserialize<T>(_moduleSerializer.Serialize(value));
```

**Separate Database Schemas Per Module:**
```csharp
// src/Modules/Agendas/Confab.Modules.Agendas.Infrastructure/EF/AgendasDbContext.cs
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.HasDefaultSchema("agendas");
}
```

**7 Well-Defined Modules:**
- Agendas, Attendances, Conferences, Saga, Speakers, Tickets, Users
- Each has its own namespace: `Confab.Modules.*`
- Each has internal layering (Api/Application/Domain/Infrastructure or Core)

### What This Architecture Is NOT

- **NOT Microservices**: The README explicitly states this is a modular monolith. All modules deploy together through the Bootstrapper. The `APIGateway` and `Services/Tickets` projects exist for future microservices transition demonstration, not current deployment.

- **NOT Just Layered**: While modules have internal horizontal layers (Api/Application/Domain/Infrastructure), the **primary architectural driver is vertical module boundaries** with enforced isolation. Horizontal layering is secondary.

- **NOT Service-Based**: Service-Based architecture has 2-5 coarse-grained services often sharing a database. This has 7 fine-grained modules with separate schemas and stronger isolation through local contracts.

### Conclusion

Confab is a textbook example of **Modular Monolith** architecture with strong module boundary enforcement through:
1. Explicit `IModule` contracts
2. Dynamic loading with enable/disable capability
3. Separate database schemas per module
4. Local contracts approach for inter-module communication (no shared types)
5. Serialization-based type translation preventing direct dependencies

The DDD patterns (aggregates, domain events, entities with business logic) are applied **within modules**, making DDD a complementary secondary style supporting the modular monolith primary architecture.
