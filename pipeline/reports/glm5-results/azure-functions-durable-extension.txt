## Analysis

This repository is the **Azure Functions Durable Extension** - a plugin that extends the Azure Functions runtime with stateful orchestration capabilities. The architecture is fundamentally a **Microkernel (Plugin)** pattern, with secondary Hexagonal Architecture characteristics for its storage abstraction.

### What This Is NOT

- **NOT Layered Architecture**: There's no horizontal layering (presentation/business/data). This is a plugin library with vertical extension points.

- **NOT Microservices**: This is a single extension library deployed as a NuGet package, not multiple independently deployable services.

- **NOT Event-Driven Architecture**: While Durable Functions orchestrates via internal events, the extension itself is structured as a plugin system with direct invocations, not an event bus pattern.

- **NOT CQRS**: No separate read/write models in the extension architecture.

- **NOT Domain-Driven Design**: Not organized around bounded contexts or aggregates - this is infrastructure/plugin code.

- **NOT Serverless Architecture**: This *enables* serverless orchestrations but is itself a plugin library, not a serverless deployment pattern.

### Summary

The repository implements a **Microkernel (Plugin)** architecture where:
- The **core** (`DurableTaskExtension`) provides the orchestration engine, function registration, and coordination
- **Plugin contracts** are defined through Azure Functions extension interfaces (`IExtensionConfigProvider`, `IWebJobsStartup`, `WorkerExtensionStartup`)
- **Storage adapters** plug in via `IDurabilityProviderFactory` 
- **Language workers** adapt via shim classes
- **User functions** are discovered and registered dynamically through trigger attributes

The Hexagonal pattern for storage abstraction (ports and adapters for Azure Storage, MSSQL, Netherite) reinforces the plugin nature, allowing the core orchestration logic to remain independent of storage implementation.
