## Architecture Classification Analysis

### Primary Style: **Layered Architecture**

The Letta repository exhibits a clear **horizontal layered architecture** with well-defined separation of concerns:

**Presentation/Interface Layer:**
- `letta/server/rest_api/routers/` — FastAPI REST endpoints organized by resource (agents, blocks, messages, tools, etc.)
- `letta/server/ws_api/` — WebSocket API for real-time streaming
- `letta/cli/` — Command-line interface
- `letta/interfaces/` — Streaming interface abstractions for different LLM providers (Anthropic, OpenAI, Gemini)

**Service/Business Logic Layer:**
- `letta/services/` — Core business services: `AgentManager`, `MessageManager`, `BlockManager`, `PassageManager`, `GroupManager`, `FileManager`, `ToolManager`, etc.
- Services orchestrate domain operations and encapsulate business rules

**Domain/Agent Layer:**
- `letta/agents/` — Agent implementations (`LettaAgent`, `LettaAgentV2`, `LettaAgentV3`, `VoiceAgent`, `EphemeralAgent`)
- `letta/groups/` — Multi-agent coordination patterns
- `letta/functions/` — Tool/function schema generation and parsing

**Data Access Layer:**
- `letta/orm/` — SQLAlchemy ORM models for persistence (Agent, Block, Message, Group, etc.)
- `letta/schemas/` — Pydantic data transfer objects

**Infrastructure/Adapters Layer:**
- `letta/llm_api/` — LLM client adapters for multiple providers
- `letta/services/mcp/` — Model Context Protocol client implementations
- `letta/adapters/` — Request/response adapters

### What This Is NOT

**NOT Microservices:**
- Single deployable artifact (one Dockerfile, one `letta_server` container)
- Single monolithic codebase (`pyproject.toml` defines one package)
- Single shared PostgreSQL database with pgvector

**NOT Event-Driven:**
- No message broker configuration in `compose.yaml`
- Synchronous REST API as primary interface
- Direct method calls between layers

**NOT Domain-Driven Design:**
- No bounded context separation
- No aggregate/value object patterns in domain model
- Layered organization by technical concern, not domain boundaries

**NOT Multi-Agent Architecture:**
- While the system manages AI agents, the repository structure itself is not organized around autonomous agent coordination — the agent classes are domain objects within a layered system
