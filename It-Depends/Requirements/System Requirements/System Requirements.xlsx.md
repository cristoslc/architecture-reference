## System Requirements (RQ)

| | Criticality | User Interface (IM) |  | 
| | --- | --- | --- | 
| |  | SR-UI-001 | Web and Mobile-Based with back-end processing and reporting and analytics that tracks site activity, connections | 
| | must have | SR-UI-002 | End-Use Ease of Use is a hard requirement | 
| |  |  |  | 
| | Criticality | Interaction Manager (IM) |  | 
| | must have | SR-IM-001 | Community members must not be able to register multiple devices to a particular email address. | 
| | must have | SR-IM-002 | EU Regulation, ability to delete profile | 
| | must have | SR-IM-003 | The ability to opt in to and out of push notifications  | 
| | must have | SR-IM-004 | Location Tracking  | 
| | must have | SR-IM-005 | Officers location must automatically shut off after 15 minutes  | 
| | must have | SR-IM-006 | Users can only have one connection per officer per 24 hour period.  | 
| |  | SR-IM-007 | A connection can  happen if the community member and the police officer are within a 10 ft from each other, | 
| | must have | SR-IM-008 | Community members find the officers that are willing to connect, officers cannot find the community members.   | 
| |  | SR-IM-009 | Officers and community members must have the ability to connect with each other without being found  | 
| |  | SR-IM-010 | Allow Hey, Blue! to track all participation by zip code.  | 
| | must have | SR-IM-011 | The App must protect user data and consider what data is stored and what data is transient. | 
| |  |  |  | 
| | Criticality | Donation and Redemption (DR) |  | 
| | must have | SR-DR-001 | The Platform must provide a way to allow Civilians to find retail establishments where they can redeem their points. | 
| | must have | SR-DR-002 | The Platform must provide a way to allow Officers to find Charities where they can gift their points. | 
| | must have | SR-DR-003 | Allow Businesses/Charities to create a storefront on the app to encourage users to redeem or donate their points. | 
| | must have | SR-DR-004 | A intrinsic reward system created (Think: confetti falling, bells and whistles happening, trophies awarded) We also uploaded a cool quote about connection to be used for each time a connection is made) (Hard Requirement) | 
| | must have | SR-DR-005 | Businesses should have a catalog of items that they are offering and the point value assigned to each item available. | 
| |  | SR-DR-006 | Allow local municipalities to offer points for fines/fees in order to get citizens back on the road (suspended licenses or to pay down any municipal fine) | 
| |  | SR-DR-007 | EX:  User gets points for connection. User applies points for product i.e.: new shoes. Shoes before Hey, Blue! Points $100. Shoes after utilizing 10 Hey, Blue! points: $75 | 
| |  | SR-DR-008 | User pays $75 of which Hey, Blue! Receives an affiliate marketing fee from retailer of 5% ($3.75)  After admin costs: $1,   $2.75 goes back into the community that participated (zip code) for connection events for more officer/ community connections, resources for the overall safety of the community (hire more officers for that particular zip code, etc. | 
| |  | SR-DR-009 | The platform must establish a way to incentivize and create connections between police and members of the community through virtual handshakes and community events | 
| |  |  |  | 
| | Criticality | Analytics and Reporting (AR) |  | 
| |  | SR-AR-001 | Analytics can take app data and aggregate it with other available public data to build comprehensive reports to show impact of app in community | 
| |  | SR-AR-002 | The ability for aggregated data to be automatically shared with local media outlets. | 
| |  | SR-AR-003 | A way to track how the points were used for each user (civilian, officer, business, municipality) | 
| | must have | SR-AR-004 | Community members to be able to see how many connections each participating officer has made.  | 
| | must have | SR-AR-005 | Tracking engagement  | 
| |  | SR-AR-006 | At the end of each month, users are notified via zip code of their participation and amount of money raised with an option for public input as to how the money will be spent. (Hey, Blue! Chooses 4 different ways to participants to vote) | 
| |  |  |  | 
| | Criticality | Media Manager (MM) |  | 
| | must have | SR-MM-001 | Uploading to social media is a Hard Requirement | 
| |  | SR-MM-002 | Connectivity to Users Social Media is an option |
